`BrokerMessage<T>` is the public message type in Broccoli. Every published and consumed message is wrapped in this envelope, which carries your typed payload alongside queue metadata generated by the broker. You interact with `BrokerMessage<T>` directly in process handlers, acknowledgement calls, and rejection calls.

## Type Parameters

The single type parameter `T` is constrained to `Clone + serde::Serialize + serde::de::DeserializeOwned`. When a message is stored in the queue the payload is serialized to JSON via `serde_json::to_string`. On consumption the JSON string is deserialized back into `T` via `serde_json::from_str`. Any type that round-trips through `serde_json` correctly is a valid payload.

<Note>
  If deserialization fails during consumption, Broccoli returns a `BroccoliError::Broker` containing the underlying parse error rather than a `BroccoliError::Serialization`. Make sure your payload type's schema is stable across producer and consumer deployments.
</Note>

## Struct Fields

<ResponseField name="task_id" type="uuid::Uuid" required>
  Unique identifier for the message. A UUID v4 value assigned at publish time by `BrokerMessage::new`. This ID is preserved throughout the message lifecycle — across retries and moves between the main, processing, and failed queues.
</ResponseField>

<ResponseField name="payload" type="T" required>
  The user-defined message content. Serialized to a JSON string when stored and deserialized back to `T` when consumed. The payload type must implement `Clone + Serialize + DeserializeOwned`.
</ResponseField>

<ResponseField name="attempts" type="u8" required>
  Number of processing attempts made so far. Starts at `0` when the message is first published and is incremented each time the message is rejected and re-queued. Once `attempts` exceeds the broker's configured `retry_attempts` limit the message is moved to the failed queue.
</ResponseField>

<ResponseField name="disambiguator" type="Option<String>">
  Optional fairness routing key set at publish time. When present, the broker routes the message through a per-disambiguator sub-queue so that no single tenant can starve others. `None` means the message belongs to the shared main queue.
</ResponseField>

## Constructor

```rust
impl<T: Clone + serde::Serialize> BrokerMessage<T> {
    pub fn new(payload: T, disambiguator: Option<String>) -> Self
}
```

<ParamField path="payload" type="T" required>
  The message content to wrap. Stored as-is in the `payload` field; serialized to JSON only when the message is passed to an underlying broker.
</ParamField>

<ParamField path="disambiguator" type="Option<String>">
  Optional fairness routing key. Pass `Some("tenant-id".to_string())` to enable per-tenant fairness queuing, or `None` for the shared queue.
</ParamField>

`BrokerMessage::new` generates a fresh UUID v4 for `task_id` and sets `attempts` to `0`. This constructor is called internally by `publish` and `publish_batch`; you rarely need to call it directly in application code.

## Usage in Handlers

The closure passed to `process_messages` receives a fully-deserialized `BrokerMessage<T>`. All four public fields are available:

```rust
use broccoli_queue::brokers::broker::BrokerMessage;

queue.process_messages("jobs", None, None, |msg: BrokerMessage<JobPayload>| async move {
    println!(
        "id={} payload={:?} attempts={} tenant={:?}",
        msg.task_id, msg.payload, msg.attempts, msg.disambiguator
    );
    Ok(())
}).await?;
```

<Tip>
  Use `msg.attempts` to implement progressive back-off logic inside your handler — for example, skip expensive external calls on the first attempt and only escalate on subsequent retries.
</Tip>

## Conversion to InternalBrokerMessage

`BrokerMessage<T>` implements `From<BrokerMessage<T>> for InternalBrokerMessage` (and `From<&BrokerMessage<T>>`). The conversion serializes the payload to a JSON string and converts the `task_id` UUID to a plain `String`:

```rust
// Implemented automatically — no user action required
impl<T: Clone + serde::Serialize> From<BrokerMessage<T>> for InternalBrokerMessage {
    fn from(msg: BrokerMessage<T>) -> Self { /* ... */ }
}
```

<Warning>
  `InternalBrokerMessage` is not part of the public API. It exists solely to pass messages between the `BroccoliQueue` façade and the concrete broker implementations (Redis, RabbitMQ, SurrealDB). Do not depend on its fields or structure in application code.
</Warning>

## Acknowledgement and Rejection

`queue.acknowledge(topic, msg)` and `queue.reject(topic, msg)` both consume (`move`) the `BrokerMessage<T>`. If you need to use the payload after the call, clone it before passing the message:

```rust
let payload = msg.payload.clone();
queue.acknowledge("jobs", msg).await?;
// payload is still available here
process_result(payload).await;
```

Similarly for rejection:

```rust
let task_id = msg.task_id;
queue.reject("jobs", msg).await?;
tracing::warn!(task_id = %task_id, "message rejected");
```

## QueueStatus

`QueueStatus` is returned by `queue.get_queue_status(queue_name, disambiguator)` and provides a snapshot of queue depth at a given moment in time.

<Note>
  `QueueStatus`, `QueueType`, and the `get_queue_status` method require the `management` Cargo feature. Enable it in your `Cargo.toml`:

  ```toml
  broccoli_queue = { version = "*", features = ["management"] }
  ```
</Note>

### QueueStatus Fields

<ResponseField name="name" type="String" required>
  The name of the queue as passed to `get_queue_status`.
</ResponseField>

<ResponseField name="queue_type" type="QueueType" required>
  The logical role of this queue snapshot. See [`QueueType`](#queuetype) below.
</ResponseField>

<ResponseField name="size" type="usize" required>
  Number of messages currently waiting in the main queue, ready to be consumed.
</ResponseField>

<ResponseField name="processing" type="usize" required>
  Number of messages currently in the processing queue — consumed by a worker but not yet acknowledged or rejected.
</ResponseField>

<ResponseField name="failed" type="usize" required>
  Number of messages that exhausted all retry attempts and were moved to the failed queue.
</ResponseField>

<ResponseField name="disambiguator_count" type="Option<usize>">
  For fairness queues, the number of active disambiguator sub-queues. `None` for standard (non-fairness) queues.
</ResponseField>

### QueueType

`QueueType` is a `Display`-enabled enum that identifies which logical queue a `QueueStatus` represents:

| Variant | Display string | Description |
|---|---|---|
| `QueueType::Main` | `"main"` | The primary inbound queue where new messages land |
| `QueueType::Processing` | `"processing"` | The in-flight queue holding messages under active processing |
| `QueueType::Failed` | `"failed"` | Messages that exceeded the retry limit |
| `QueueType::Fairness` | `"fairness"` | A per-disambiguator sub-queue used for tenant fairness |

```rust
use broccoli_queue::brokers::management::{QueueStatus, QueueType};

let status = queue.get_queue_status("jobs".to_string(), None).await?;
println!(
    "Queue '{}' ({}) — waiting: {}, processing: {}, failed: {}",
    status.name, status.queue_type, status.size, status.processing, status.failed
);

if status.queue_type == QueueType::Fairness {
    println!("Active disambiguators: {:?}", status.disambiguator_count);
}
```
